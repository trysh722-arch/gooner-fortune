# 오늘의 구너 운세 — 도감/컬렉션 확장 설계 v2

> 데일리 운세 가챠 → **수집 → 베스트11 등록 → 내 운세 강화** 루프로 확장.
> 저장은 **하이브리드(localStorage + Supabase/카카오 로그인)**. 호스팅은 GitHub Pages 정적 그대로(운영 서버 없음).
> **개발은 라이브 `index.html` 안 건드리고 별도 테스트 페이지(`/dev/`)에서 → 확인 후 배포.**
> v2 변경: 퀴즈 코어 제외, 카드 버전(변형) 도입, 포메이션 선택형, 중복=카운트만, 단계 재정렬.

---

## 0. 코어 루프 (매일 들어올 이유)

```
입장 → 오늘의 카드 뽑기(가챠 연출) → 운세(스쿼드 부스트 반영)
     → 도감에 수집(+1) → 베스트11 등록/교체
     → (한정 카드는 특정일에만) → 친구랑 도감·스쿼드 비교/자랑
```

- **가챠(도파민) → 수집(목표) → 스쿼드(성장) → 운세강화(보상) → 공유(확산)** 가 하나로 물림.
- **데일리 퀴즈는 코어에서 제외**(입장 게이트 X). 원하면 선택 탭 + 보상 없음(§4).

---

## 1. 카드 획득 규칙

- **하루 1장**: 그날의 "오늘의 럭키 거너" 카드가 자동으로 도감에 들어감. (지금 뽑기 로직 그대로, 결과만 영구 저장)
- **중복 = 꽝**: 이미 가진 카드(같은 버전)가 또 뽑히면 **도감에 새로 안 들어감(꽝)**. 카운트도 없음 — 가챠 특유의 꽝 손맛. (단 그날 럭키 거너로는 정상 표시, 운세는 작동). **다른 버전은 별개 카드라 꽝 아님**(§5). 후반 꽝 누적은 한정/이벤트 카드로 새 수집거리 공급해 메움.
- **한정 카드**: 특정 이벤트일에만 풀에 등장(§5).

> 채우는 속도 = 기본 1장/일 → 도감 완성은 장기 목표 = 장기 리텐션. 한정/이벤트로 가속.

---

## 2. 도감 (컬렉션)

- 전체 카드 그리드. **획득=풀컬러 / 미획득=실루엣**(채우고 싶게).
- 섹션: 현역 / 레전드 / 감독 / **한정**(제일 빛나는 칸).
- 카드별: 등급색, 보유 카운트, 최초 획득일.
- 진척도 바: "전체 OO장 중 OO장 수집 (OO%)".
- 짜치 방지: 다크+골드 프리미엄 톤, 미획득 실루엣, 한정칸 골드 테두리.
- **규모**: 고유 선수 23은 기본일 뿐 → **카드 버전(에디션, §5) + 레전드 대거 확장 + 한정카드**로 **100~150장+** 목표. 도감 완성이 장기 목표가 되게. (23만이면 금방 끝나 동기 약함)

## 3. 베스트11 + 포메이션(선택형) → 운세 부스트

- **포메이션 선택형** — 4-3-3 / 4-2-3-1 / 4-4-2(무패 클래식) / 4-1-4-1 / 3-4-3 / 3-5-2 / 4-3-2-1 등 다양. 고른 뒤 모은 카드를 포지션 슬롯에 배치. (더 추가 가능)
- 카드 **OVR**: 등급 기반 — ultra 99 / legend 92 / manager 특수 / star 85 / standard 78.
- **케미스트리**: 인접·동포지션·동등급 링크가 빛남. 풀케미일수록 보너스.
- **스쿼드 파워** = 평균 OVR × 케미 배수 → 우상단 큰 숫자(FIFA OVR 느낌).
- **감독은 필드 선수 아님 → 전용 '감독 슬롯'**(필드 11명과 별개 1자리). 아르테타/벵거 장착 시 **팀 전체 버프**(포지션 차지 X). 가챠로 감독 카드가 떠도 수집은 그대로, 스쿼드에선 감독 슬롯에만 장착. (옵션: 아르테타는 실제 선수였으니 "선수 시절" 버전 별도 가능 / 벵거는 감독 전용)
- **운세 연동(핵심)**: 스쿼드 파워가 그날 **승률·폼에 보너스**(예: +0~+8%p, 상한 둠). 운세는 여전히 매일 랜덤, 스쿼드는 "엣지"만 — 밸런스 유지.
- 빈 슬롯은 실루엣 + "이 자리 채우면 +X%" 유도.

## 4. 데일리 퀴즈 (코어 제외 · 선택 · 후순위)

- 입장 게이트로 두지 않음(첫 화면 퀴즈 X). **선택 탭 '오늘의 구너 상식'**으로 분리.
- **게임 보상 없음**(추가뽑기·확률업 X). 순수 재미 + 연속정답 스트릭 배지 정도.
- 날짜 시드로 전원 같은 문제(공유·비교용). 문제 은행은 **팩트체크 후** 등록.
- 우선순위 후순위(P5) — 코어(수집·스쿼드) 먼저.

## 5. 한정/이벤트 카드 + 카드 버전(변형)

- **카드 = 선수 × 버전.** 버전은 **시즌·업적 에디션**(특정 시즌/순간의 그 선수) → 도감에서 각각 다른 칸.
  - 예: 사카[기본] / 사카[23-24 시즌 스페셜] / 앙리[무패 시즌 INVINCIBLE]
- **버전 차이는 디자인으로 한눈에 구분(★중요)**: 프레임·색·배지·연도 다르게.
  - 기본(현역 화이트/블랙) / 무패(딥 골드 + "INVINCIBLE 03-04" 배지) / 시즌 스페셜(시즌 색 + 연도 배지) / TOTS 등.
- **이벤트 캘린더**(날짜 → 특별 버전 풀): 매치데이·북런던더비·유럽경기·우승·시즌 기념. 평소엔 기본, 이벤트일엔 한정 버전 등장 → 기간 지나면 못 얻음(FOMO).
- **같은 선수 여러 버전 모으기 = 도감 깊이의 핵심**.
- ⚠️ **버전-선수 매칭은 사실대로**: 무패 시즌(03-04) 버전은 그 시즌 실제 선수만(앙리·베르캄프·비에이라·융베리·콜로투레 등). **사카는 2001년생 → 무패 시즌 없음 → "최근 시즌 스페셜"로**. 팬이 바로 잡아냄.

## 6. 저장 구조 (하이브리드)

### 게스트(로그인 X) — localStorage
- `collection`(카드별 count·최초획득일), `squad`(포메이션·슬롯), `progress`(마지막 뽑기 날짜 등) 로컬 저장. 로그인 없이 바로 수집 시작.

### 로그인 시 — Supabase (무료 BaaS, GitHub Pages 정적 유지)
- **Auth**: 카카오 OAuth (한국 유저 마찰 최소).
- **테이블**: `profiles` · `cards_owned`(user_id, card_id, count, first_obtained) · `squads`(formation, slots jsonb) · `progress`. RLS로 본인 데이터만 접근.
- **동기화**: 로그인 순간 **localStorage ↔ 클라우드 합집합 머지**(count=max, 최초획득일=이른 날). 이후 뽑기마다 로컬+클라우드 동시 기록. 스쿼드는 최신 우선.

### 보안 (별 이슈 없음 — 전제만 지키면)
- localStorage: 그냥 로컬 데이터, 민감정보 0 → 이슈 없음.
- Supabase: **공개키(anon)만 클라에 노출**(원래 노출 설계) + **RLS 필수**(본인 데이터만) + **service_role 키 절대 클라 금지** + 도메인 화이트리스트.
- 저장 정보 = 닉네임+도감 수준(민감X), 비번은 카카오가 처리.
- 결제·민감정보 없는 팬 재미툴이라 위험 낮음. 카카오 닉/이메일 받으면 나중에 개인정보 안내 한 줄 권장.
- 치팅 주의: 뽑기가 클라이언트 결정형 → localStorage 조작 여지. 팬툴이라 저위험·허용. **랭킹 넣으면** 그땐 서버에서 뽑기 결과 발급으로 보강.

### 안 바뀌는 것
- 프론트 = 기존 단일 HTML 확장(Vanilla + Supabase JS SDK CDN). **GitHub Pages 그대로, 운영 서버 0.**

## 7. 화면 구성

1. **입장**(티켓) — 기존
2. **오늘**: 가챠 뽑기 연출 → 운세(승률·4운세, 스쿼드 보너스 표시) → "도감 +1"
3. **도감**: 그리드(획득/실루엣)·섹션·진척도·한정칸
4. **스쿼드**: 포메이션(선택형) 피치·OVR·케미·오늘 부스트
5. **마이**: 로그인/백업·통계
6. **공유**: 내 스쿼드/도감 OG 카드
7. (선택) **구너 상식**: 데일리 퀴즈

## 8. 단계별 출시 (테스트 → 배포 / 백엔드 없는 것부터)

> 라이브 `index.html` 안 건드리고 **별도 테스트 페이지(`/dev/`)** 에서 만들고, OK 나면 라이브 승격.

- **P1. 도감 + 수집(localStorage)** — 매일 뽑은 카드 쌓이고 도감에서 보기. [백엔드 X · 먼저 테스트]
- **P2. 베스트11 + 포메이션(선택형) + 스쿼드 부스트** — 수집 동기 완성. [백엔드 X]
- **P3. 카드 버전(변형) + 한정/이벤트 카드** — 도감 깊이·FOMO. [백엔드 X]
- **P4. Supabase + 카카오 로그인 + 클라우드 백업/머지** — "날아갈 걱정" 해소.
- **P5(선택·후순위). 데일리 퀴즈(보상 없음) + 랭킹.**

> P1~P3는 백엔드 없이 로컬에서 다 테스트 가능 → 반응 보고 P4에서 클라우드.

## 9. 확정된 결정 / 남은 것

- **확정**: 포메이션 **선택형(다양)** · 중복 = **꽝(카운트 없음)** · 퀴즈 = **선택 탭·보상 없음·후순위** · 한정 = **카드 버전(시즌/업적 에디션, 디자인 명확 구분)** · 개발 = **테스트페이지 먼저, 라이브 분리**
- **남은 것(P4 때)**: 카카오 디벨로퍼 앱키 + Supabase 프로젝트(둘 다 무료·사용자 계정) · OVR/부스트 상한 수치

## 10. 리스크 / 메모

- 단일 HTML이 커지고 있음 → P2~P3쯤 파일 분리(모듈화) 검토.
- 한정카드/이벤트 = 지속 콘텐츠 운영 비용. 이벤트 캘린더 미리 비축.
- 카드 버전 도입 = 선수마다 버전별 비주얼/누끼 필요(한정 버전 추가 시 디자인 작업).
- 저작권: 선수 사진·구단 마크는 기존과 동일(비영리 유지 전제).
